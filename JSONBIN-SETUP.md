# 🔧 راهنمای تنظیم JSONBin.io برای همگام‌سازی داده‌ها

## مشکل فعلی
داده‌ها در `localStorage` مرورگر ذخیره می‌شوند و بین دستگاه‌های مختلف (گوشی و دسکتاپ) مشترک نیستند.

## راه حل: استفاده از JSONBin.io

JSONBin.io یک سرویس رایگان برای ذخیره JSON در cloud است که داده‌های شما را بین همه دستگاه‌ها همگام می‌کند.

---

## 📝 مراحل تنظیم (5 دقیقه)

### مرحله 1: ساخت حساب کاربری

1. به **https://jsonbin.io** بروید
2. روی **"Sign Up"** کلیک کنید
3. با ایمیل یا GitHub ثبت‌نام کنید (رایگان است)

### مرحله 2: دریافت API Key

1. بعد از ورود، به **Dashboard** بروید
2. در بخش **"API Keys"**، روی **"Create API Key"** کلیک کنید
3. یک نام برای API Key انتخاب کنید (مثلاً: `atiyeh-form`)
4. **API Key** را کپی کنید (مثل: `$2b$10$xxxxxxxxxxxxxxxxxxxxxxxxxxxxx`)

⚠️ **مهم**: این API Key را در جای امن نگه دارید!

### مرحله 3: تنظیم در پروژه

1. فایل `config.js` را باز کنید
2. API Key را در قسمت `JSONBIN_API_KEY` وارد کنید:

```javascript
const JSONBIN_API_KEY = '$2b$10$xxxxxxxxxxxxxxxxxxxxxxxxxxxxx'; // API Key خود را اینجا وارد کنید
const JSONBIN_BIN_ID = ''; // این را خالی بگذارید - به صورت خودکار ایجاد می‌شود
```

3. فایل را ذخیره کنید

## ✅ تمام! 

حالا:
- ✅ داده‌ها به صورت خودکار در JSONBin.io ذخیره می‌شوند
- ✅ Bin ID به صورت خودکار ایجاد و ذخیره می‌شود
- ✅ داده‌ها بین همه دستگاه‌ها (گوشی، دسکتاپ، تبلت) همگام می‌شوند
- ✅ وقتی در پنل ادمین وارد می‌شوید، همه داده‌ها از cloud بارگذاری می‌شوند

---

## 🔄 نحوه کار

1. **هنگام ثبت نام**: داده‌ها هم در localStorage و هم در JSONBin.io ذخیره می‌شوند
2. **هنگام ورود به پنل ادمین**: داده‌ها از JSONBin.io بارگذاری می‌شوند
3. **بین دستگاه‌ها**: همه داده‌ها از یک Bin مشترک خوانده/نوشته می‌شوند

---

## ⚠️ نکات مهم

- **API Key را محرمانه نگه دارید** - این Key دسترسی کامل به داده‌های شما دارد
- **Bin ID به صورت خودکار ایجاد می‌شود** - نیازی به وارد کردن دستی نیست
- **داده‌ها هم در localStorage و هم در cloud ذخیره می‌شوند** - اگر JSONBin.io در دسترس نباشد، از localStorage استفاده می‌شود
